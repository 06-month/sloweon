## 1. IA 설계 원칙

- 고객 IA는 상품 탐색, 사이즈 판단, 장바구니, 주문 완료까지의 흐름을 최우선으로 둔다.
- 관리자 IA는 오늘 처리해야 할 운영 업무, 상품/재고 관리, 주문/배송/교환/반품 처리를 최우선으로 둔다.
- 로그인 사용자, 비회원 사용자, 관리자 사용자의 접근 권한을 명확히 분리한다.
- 패션몰 특화 정보인 핏, 색상, 사이즈, 실측표, 모델 착용 정보, 룩북, 코디 세트, 재입고 알림이 탐색 구조에서 빠지지 않아야 한다.

## 7. 1차 IA 출시 범위

1차 출시에서는 고객 구매 흐름과 운영 필수 화면을 우선 구현한다.

- 고객: 홈, 상품 목록, 상품 상세, 룩북, 장바구니, 주문서, 결제 결과, 로그인/회원가입, 마이페이지, 주문 상세
- 관리자: 로그인, 대시보드, 상품 관리, 옵션/재고 관리, 사이즈표 관리, 주문/배송 관리
- 시스템: 인증 API, 상품 API, 장바구니 API, 주문 API, 결제 준비/검증 API, 관리자 API
