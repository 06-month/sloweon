## 1. 제품 개요

### 1.1 제품명

남성 컨템포러리 패션 쇼핑몰

### 1.2 제품 목적

20대부터 30대 남성 고객이 일상에서 입기 좋은 컨템포러리 의류를 빠르게 탐색하고, 자신의 체형과 취향에 맞는 핏과 사이즈를 확인한 뒤 안정적으로 구매할 수 있는 패션 이커머스 서비스를 구축한다. 운영자는 시즌 드롭, 룩북, 코디 세트, 상품/옵션/재고, 주문/배송/교환/반품을 한 곳에서 관리할 수 있어야 한다.

### 1.3 핵심 목표

- 남성 컨템포러리 패션 상품을 스타일, 핏, 사이즈, 코디 기준으로 쉽게 탐색하게 한다.
- 상품 상세에서 모델 착용 정보, 실측 사이즈, 핏 가이드, 코디 상품을 명확히 제공한다.
- 상의, 하의, 아우터, 셋업, 신발, 액세서리의 옵션/재고/교환 정책을 정확하게 관리한다.
- 장바구니부터 결제까지의 구매 흐름을 모바일 기준으로 빠르고 안정적으로 제공한다.
- 관리자에게 시즌 상품 등록, 룩북 구성, 코디 세트 구성, 재입고 요청 관리, 주문/배송/CS 처리를 제공한다.

### 1.4 주요 사용자

| 사용자 | 설명 | 핵심 니즈 |
|---|---|---|
| 스타일 입문 남성 | 옷 조합에 익숙하지 않은 고객 | 코디 추천, 기본템 큐레이션, 쉬운 사이즈 선택 |
| 캐주얼 데일리 고객 | 출근/등교/일상복을 찾는 고객 | 편한 핏, 합리적 가격, 빠른 배송 |
| 트렌드 얼리어답터 | 세미오버·릴랙스 실루엣, 신상품 드롭 선호 | 신상품, 드롭, 룩북, 스타일 태그 |
| 재구매 고객 | 브랜드 핏과 사이즈를 알고 있는 고객 | 빠른 재구매, 재입고 알림, 포인트/쿠폰 |
| 쇼핑몰 운영자 | 상품/주문/콘텐츠 운영 담당 | 상품 등록, 재고 관리, 룩북 관리, 주문 처리 |

### 1.5 고객 페르소나

핵심 고객 페르소나는 [고객 페르소나 문서](./customer-personas.md)를 기준으로 한다. 디자인, 개발, QA, 런칭 단계에서 최소 3개 이상의 페르소나 관점으로 주요 화면과 기능을 검토한다.

| 페르소나 | 고객 유형 | 핵심 검토 관점 |
|---|---|---|
| 김도현 | 스타일 입문 남성 | 코디 추천, 쉬운 선택, 첫 구매 전환 |
| 박준서 | 데일리 직장인 | 빠른 탐색, 신뢰, 배송/교환 정책 |
| 이태오 | 트렌드 얼리어답터 | 신상품 드롭, 룩북, 재입고 알림 |
| 최민재 | 사이즈 민감 재구매 고객 | 실측표, 착용 리뷰, 사이즈 교환 |

## 2. 제품 범위

### 2.1 포함 범위

- 회원가입, 로그인, 로그아웃, 비밀번호 재설정, 세션 관리, 비회원 주문, 마이페이지
- 남성 패션 카테고리, 상품 목록, 상품 상세, 검색, 필터, 정렬
- 상품 옵션, 색상, 사이즈, 핏, 실측 사이즈, 모델 착용 정보
- 룩북, 스타일 태그, 코디 세트, 함께 입기 좋은 상품
- 장바구니, 바로구매, 주문서, 결제, 주문 조회
- 쿠폰, 포인트, 재입고 알림, 찜하기
- 배송지, 배송비, 송장번호, 배송 상태
- 주문 취소, 사이즈 교환, 반품, 환불
- 리뷰, 착용 리뷰, 사진 리뷰, 상품 문의
- 관리자 상품/옵션/재고/룩북/코디/주문/배송/회원/CS 관리
- 통계, 감사 로그, 알림, 분석 이벤트

### 2.2 1차 출시 제외 범위

- 입점사 마켓플레이스
- 해외 배송
- 정기구독 결제
- AI 기반 자동 스타일 추천
- 중고거래
- 라이브커머스
- 오프라인 매장 재고 연동

## 17. 핵심 정책

### 17.1 가격 정책

- 최종 주문 금액은 서버 계산값을 기준으로 한다.
- 상품 가격은 결제 시점의 가격을 주문 항목에 스냅샷으로 저장한다.
- 할인, 쿠폰, 포인트, 배송비는 주문서 진입과 결제 직전에 재계산한다.

### 17.2 인증/세션 정책

- 회원 비밀번호는 평문 저장을 금지하고 안전한 단방향 해시로 저장한다.
- 로그인 성공 시 사용자 권한, 계정 상태, 세션 만료 시각을 명확히 관리한다.
- 로그아웃 시 현재 세션 또는 리프레시 토큰을 즉시 무효화한다.
- 비밀번호 재설정, 이메일/휴대폰 인증 토큰은 해시로 저장하고 만료 시간을 둔다.
- 관리자 계정은 일반 회원과 동일 테이블을 사용할 수 있으나 권한과 접근 경로를 분리한다.
- 관리자 로그인은 가능하면 2단계 인증 또는 IP 제한을 적용한다.
- 비회원 장바구니와 주문은 guestSessionId 또는 guestOrderKey로 관리하되 개인정보 접근은 제한한다.

### 17.3 옵션/재고 정책

- 재고는 색상과 사이즈 조합 단위로 관리한다.
- 결제 성공 시 재고를 차감한다.
- 취소, 결제 실패, 반품 완료 시 정책에 따라 재고를 복구한다.
- 품절 옵션은 구매 버튼을 비활성화하고 재입고 알림 신청을 제공한다.

### 17.4 사이즈 정보 정책

- 모든 상품(의류/신발/가방 포함)은 상품군에 맞는 실측 사이즈표를 필수로 가진다.
- 상품군별 실측 항목은 다를 수 있다. SizeSpec.categoryGroup(TOP/BOTTOM/OUTER/SETUP/SHOES/BAG/ACCESSORY) 값에 따라 사용하는 필드가 다르며, 신발은 footLength/footWidth, 가방은 bagWidth/bagHeight/bagDepth를 사용하고 해당 없는 필드는 비워둔다.
- 모델 착용 정보와 구매자 착용 리뷰는 사이즈 판단 보조 정보로 제공한다.
- 실측 오차 범위와 측정 방식 안내를 상세 페이지에 제공한다.

### 17.5 교환/반품 정책

- 사이즈 교환은 배송 완료 후 정해진 기간 내 요청 가능하도록 한다.
- 착용 흔적, 세탁, 택 제거 등 교환/반품 불가 조건을 명확히 안내한다.
- 교환 요청 시 대상 옵션의 재고를 확인한다.
- 반품 사유는 통계에 반영해 상품 상세, 사이즈표, 품질 개선에 활용한다.

### 17.6 룩북/코디 정책

- 룩북에는 하나 이상의 착용 상품이 연결되어야 한다.
- 품절 상품이 포함된 코디 세트는 품절 또는 대체 상품 안내를 제공한다.
- 코디 세트 구매 시 각 상품의 색상/사이즈 옵션을 개별 선택해야 한다.

### 17.7 데이터 정합성 정책

- 주문, 결제, 재고, 쿠폰, 포인트 변경은 실패 시 롤백 가능한 구조로 처리한다.
- 결제 웹훅과 결제 승인 API는 멱등성을 보장해야 한다.
- 관리자 재고 변경과 주문 재고 차감은 모두 재고 원장에 기록한다.
- 주문 상세는 상품 수정 이후에도 결제 시점의 상품명, 색상, 사이즈, 가격을 유지한다.

## 21. 1차 출시 기준

- 남성 컨템포러리 패션몰 콘셉트가 홈, 상품 목록, 상품 상세, 룩북에 명확히 드러난다.
- 고객이 상품을 카테고리, 핏, 색상, 사이즈, 스타일 태그로 탐색할 수 있다.
- 상품 상세에서 실측 사이즈표, 모델 착용 정보, 소재/세탁 정보, 교환/반품 안내를 확인할 수 있다.
- 고객이 색상/사이즈 옵션을 선택해 장바구니와 결제를 완료할 수 있다.
- 회원가입, 로그인, 로그아웃, 비밀번호 재설정, 세션 만료 처리가 정상 동작한다.
- 결제 성공 시 주문이 생성되고 재고가 정확히 차감된다.
- DB 마이그레이션, 핵심 제약 조건, 주문/결제/재고 트랜잭션 기준이 구현되어야 한다.
- 품절 옵션은 구매할 수 없고 재입고 알림을 신청할 수 있다.
- 관리자가 상품, 옵션, 재고, 사이즈표, 룩북, 주문, 배송을 관리할 수 있다.
- 관리자 API와 화면은 권한별 접근 제어가 적용되어야 한다.
- 모바일에서 상품 탐색, 옵션 선택, 장바구니, 주문, 결제 흐름이 정상 동작한다.
- P0 요구사항이 모두 구현되고 주문/결제/재고 관련 치명적 결함이 없어야 한다.
